第二次讨论会——主题：博客搭建

DAY - 01

主要是总述+快速建立成果
希望能引出大家的兴趣

且本次流程不仅仅是搭建一个博客，也算是一个简单的搭建网站的教程，麻雀虽小五脏俱全，也是包含了一些自建网站的技术内容的。

如何实现？

内容一

  1. npm: 类似于 Java 中的 maven 或者 python 中的 pip，可以拉去网上已经构建好的第三方库，比如我们要用到的 hexo 就可以用 npm 来拉取。

  2. Hexo: 是一个可以快速构建博客的框架，且支持各种 DIY，同时社区中也提供了各种各样的主题。它可以通过几行命令快速搭建一个可用的博客网站

  3. GitHub pages: github 有个功能叫 github pages，这是一个特殊的仓库，可以将仓库中的 index.html 映射到互联网上作为一个所有人都可以访问的网站。

  4. Git: 通过 Git 工具，我们可以将本地搭建的 博客 推送到 GitHub 中的特殊仓库中。

（但以上内容无需你有任何基础，甚至不需要会任何前端相关代码，也无需会git命令操作……有脑子）

内容二

  1. Markdown语法：一种简便的记笔记的语法，旨在快速编写简洁美观的文章。
    支持markdown的软件会让你的markdown语法写的文章自动添加一些格式
  2. 图床：一种专门存图片的技术，可以将图片存入互连网，然后将其变成可访问的链接的方式。
    主要是来配合markdown语法编辑器，因为markdown不支持存图片，所以可以通过使用图床的链接来访问的形式。
    同时还可以缓解网站服务器压力
  3. DNS域名解析: 可以将我们的网站 URL 更改为我们自己的网址
  4. CDN加速：源站的资源缓存到位于全国各地的CDN节点上，用户请求资源时，就近返回节点上缓存的资源，而不需要每个用户的请求都回您的源站获取，避免网络拥塞、分担源站压力，保证用户访问资源的速度和体验。

项目文件夹介绍

  • .github: 部署相关
  • node_modules: 依赖的包
  • scaffolds: 模板，创建新的博客文章的时候，随着一起创建的一些字段

  • themes: 主题文件夹，你下载的博客主题
  • _config.landscape.yml: `默认主题的配置文件，不用管
  • _config.yml: 博客配置文件
  • 其他不用管

可能看着挺多的，我们一般只会动 source/_post, themes 和 _config.yml, 其他的可以暂时不用理

生成密钥

这一步主要是让你和你的GitHub账户真正的建立连接

然后一直按回车，就会生成一段密钥

然后这段密钥存放在(Windows)

"C:/Users(用户)/用户的名字/.ssh/id_rsa.pub"

使用各种文本编辑器打开，复制其中的密钥

使用密钥

  1. 打开你的GitHub主页，

  2. 点击右上角的头像，

  3. Settings，

  4. 左侧 SSH and GPG keys

  5. 稍微右上的位置，绿色 New SSH key

  6. Title位置随便起个名，然后将上面复制的密钥粘贴到 Key 这一栏，然后 点绿色的 Add SSH key

  7. 完成
